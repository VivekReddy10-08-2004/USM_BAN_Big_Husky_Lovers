## Problem we tackled (in plain English)

In recruiting operations, one key challenge is **understanding which roles are likely to stay open the longest** and **where hiring bottlenecks may exist**.

Our pipeline turns raw job requisition data (including messy fields like `"236 days ago"` and large blocks of job description text) into structured, analyzable features that support:

- **Segmentation:** group jobs into “complexity tiers” (fast-fill vs hard-to-fill)
- **Forecasting:** predict how long a role may remain open (`Predicted_Days_Open`)
- **Dashboarding:** export a flat file ready for Power BI consumption

---

## Dataset (quick overview)

The raw dataset includes columns such as:
- `DAYS_OPEN` (string like `"236 days ago"` in raw form)
- `PRIMARY_LOCATION`
- `WORKER_SUB_TYPE`
- `JOB_PROFILE`
- `MANAGEMENT_LEVEL_JOB_REQUISITION` (has some missing values)
- `NUMBER_OF_OPENINGS_AVAILABLE`
- `JOB_DESCRIPTION`, `JOB_PROFILE_SUMMARY` (text-heavy)

---

## How the pipeline works

### Step 1 — Clean & standardize the raw data
You have two options:

#### Option A: Basic cleaning (fast)
Use `HRcleaner.py` to:
- Parse `DAYS_OPEN` into an integer
- Handle missing `MANAGEMENT_LEVEL_JOB_REQUISITION` (drop or impute)
- Clean text fields (lowercase, remove punctuation/extra whitespace)
- Encode key categories into numeric codes for modeling
- Export:
  - `Cleaned_ML_Ready.csv`
  - `category_mappings.json`
  - `cleaning_report.json`

#### Option B: Enhanced cleaning + EDA (recommended for analysis)
Use `HRDeepClean.py` to do everything in Option A **plus**:
- Data profiling (summary stats, distributions)
- Correlations for numeric fields
- Keyword extraction from text fields (with stopwords + n-grams)
- Data quality metrics and format validation  
Exports:
- `Cleaned_ML_Ready.csv`
- `category_mappings.json`
- `cleaning_report.json`
- `data_profile.json`
- `quality_metrics.json`

---

### Step 2 — Modeling (NLP + clustering + prediction)
Notebook: `Notebooks/BAN_NLP.ipynb`

This notebook:
1. Uses TF‑IDF on cleaned job descriptions
2. Uses PCA to compress text features into 3 “themes”
3. Runs K-Means clustering into **3 complexity clusters**
4. Trains a tuned Random Forest model to predict `DAYS_OPEN`
5. Appends:
   - `Complexity_Cluster`
   - `Predicted_Days_Open`
6. Exports a flat file for dashboarding:
   - `Forecasted_HR_Data_For_PowerBI.csv` (in the notebook)
   - (repo includes `Forecasted_HR_Data_For_PowerBI (1).csv`)

---

### Step 3 — Visualization / reporting
- `hackathon.pbix` uses the exported forecasted dataset to build an executive dashboard.
- `BusinessAnalytics.py` can generate quick charts, summaries, and image outputs (PNG + CSV summaries) from any cleaned CSV.

## Recommended workflow (exact commands)

### 1) Install dependencies
```bash
pip install pandas numpy matplotlib scikit-learn
```

### 2) Run basic cleaner
```bash
python Code/HRcleaner.py "Data/Open Reqs Mar 2026_2026-04-01.csv"
```

### 3) OR run enhanced cleaner + EDA
```bash
python Code/HRDeepClean.py "Data/Open Reqs Mar 2026_2026-04-01.csv" --output-dir ./output
```

To drop null management levels instead of imputing:
```bash
python Code/HRDeepClean.py "Data/Open Reqs Mar 2026_2026-04-01.csv" --output-dir ./output --null-strategy drop
```

### 4) Run quick business analytics charts (optional)
```bash
python Code/BusinessAnalytics.py Data/Cleaned_ML_Ready.csv
```

### 5) Run modeling notebook
Open:
- `Notebooks/BAN_NLP.ipynb`

Ensure `Cleaned_ML_Ready.csv` is available in the notebook runtime (or update the path), then run all cells to generate the Power BI-ready export.

## Notes / assumptions

- `DAYS_OPEN` is converted into an integer; when missing/unparseable it may be filled with **-1** as a sentinel value (so the column stays numeric for ML).
- Missing `MANAGEMENT_LEVEL_JOB_REQUISITION` values can be:
  - **imputed** to `"Uncategorized"` (default), or
  - **dropped** if stricter quality is required.
- Category encodings are saved so codes remain interpretable in BI tools.
